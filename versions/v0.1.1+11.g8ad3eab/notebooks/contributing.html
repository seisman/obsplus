Contributing¶

Contributions to ObsPlus are welcomed and appreciated. Before proceeding please be aware of our code of conduct.

Here are a few details you will need:

Getting setup¶

The following steps are needed to setup ObsPlus for development:

1. Clone ObsPlus¶

git clone https://github.com/niosh-mining/obsplus

2. Pull tags¶

Make sure you pull all the latest git tags (especially if some time has passed since you cloned ObsPlus) so that the dynamic versioning provided by versioneer works.

git pull origin master --tags

3. Create a virtual environment (optional)¶

Create and activate a virtual environment so ObsPlus wont mess with your base (or system) python installation.

Assuming you are using Anaconda:

conda create -n obsplus_dev obspy pandas pre-commit
conda activate obsplus_dev
cd obsplus

4. Install ObsPlus in development mode¶

pip install -e .[dev]

5. Setup precommit hooks¶

ObsPlus uses several pre-commit hooks to ensure the code stays tidy. These should be simple to install like so:

pre-commit install -f

Branching and versioning¶

We just create new features or bug fixes in their own branches then merge them into master via pull requests. We may switch to a more complex branching model if the need arises but for now we are just keeping things simple.

If substantial new features have been added since the last release we will bump the minor version, if only bug fixes/minor changes have been made we bump the patch version. Like most python projects, we only loosely follow semantic versioning in that we won’t bump the major version until ObsPlus is very stable.

Running the tests¶

The tests suite is run with pytest. Simply make sure your current directory is set to the cloned ObsPlus repo, that you have followed the development setup instructions listed above, and invoke pytest from the command line:

pytest tests

Building the documentation¶

The documentation can be built using the script called “make_docs.py” in the scripts directory. As long as you followed the instructions above all the required dependencies should be installed.

python scripts/make_docs.py

You can then access the docs by double clicking on the newly created html index at docs/_build/html/index.html.

General guidelines¶

ObsPlus uses Black and flake8 for code linting. If you have properly installed ObsPlus’ pre-commit hooks these will be invoked automatically when you make a git commit. If any complaints are raised simply address them and try again. Don’t worry it happens to everyone ;)

Use numpy style docstrings. All public code (doesn’t start with a ) should have a “full” docstring but private code (starts with a) can have an abbreviated docstring.

ObsPlus makes extensive use of Python 3’s type hints. You are encouraged to annotate any public functions/methods with type hints.

Prefer pathlib.Path to strings when working with paths. However, when dealing with millions of files (eg ObsPlus’ indexers) strings may be preferred for efficiency.

Follow PEP8’s guidelines for imports.

Working with dataframes¶

Column names should be snake_cased whenever possible.

Always access columns with getitem and not getattr (ie use df['column_name'] not df.column_name).

Prefer creating a new DataFrame/Series to modifying them inplace. Inplace modifications should require opting in (usually through an inplace key word argument).
